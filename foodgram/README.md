# Фудграм

Продуктовый помощник: сайт, где пользователи публикуют рецепты, подписываются на авторов, добавляют рецепты в избранное и формируют список покупок.

## Стек технологий

- **Backend:** Django, Django REST Framework, Djoser (аутентификация), PostgreSQL
- **Frontend:** React
- **Инфраструктура:** Docker, Docker Compose, Nginx, GitHub Actions (CI/CD)

## Адрес сайта

Проект развёрнут по адресу: [foodgram.by](http://foodgram.by)

> Если домен не настроен, используйте IP сервера: `158.160.176.150`

## Функциональные возможности

- Регистрация и аутентификация пользователей (токены)
- Управление профилем (изменение аватара)
- Публикация, редактирование и удаление рецептов
- Загрузка изображений (base64)
- Тегирование рецептов
- Поиск ингредиентов по началу названия
- Добавление рецептов в избранное
- Формирование списка покупок из нескольких рецептов с суммированием ингредиентов
- Скачивание списка покупок в текстовом файле
- Подписка на авторов
- Короткие ссылки на рецепты
- Административная панель для управления контентом

## Запуск проекта локально (для разработки)

1. Клонируйте репозиторий:
   ```bash
   git clone git@github.com:mercv21/foodgram.git
   cd foodgram

2. Создайте виртуальное окружение и установите зависимости:
    cd backend
    python -m venv venv
    source venv/bin/activate  # Windows: venv\Scripts\activate
    pip install -r requirements.txt

3. Выполните миграции и загрузите ингредиенты:
    python manage.py migrate
    python manage.py load_ingredients
    python manage.py runserver

4. В отдельном терминале соберите фронтенд (или используйте контейнер):
    cd ../frontend
    npm install
    npm run build

## Развёртывание на сервере (через Docker)

1. Скопируйте на сервер папки infra, backend, frontend.

2. Установите Docker и Docker Compose.

3. В папке infra создайте файл .env с переменными (см. infra/.env.example).

4. Запустите контейнеры:
    bash
    docker compose up -d --build

## Автоматический деплой (CI/CD)

При пуше в ветку main GitHub Actions выполняет:
1. копирование кода на сервер,
2. пересборку и запуск контейнеров.

Workflow описан в .github/workflows/deploy.yml.
